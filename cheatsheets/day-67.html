Day 67 Cheat Sheet

Network & Connectivity Commands

Quick reference for viewing network interfaces, testing connectivity, resolving DNS, tracing routes, and inspecting open ports.

Interface & IP Information

CommandPurposeExample
ip addr showShow all interfaces & IPsip addr show
ip -brief addrCompact interface/IP summaryip -brief addr
ifconfigLegacy interface infoifconfig

Routing Table

ip route showFull routing table
ip route | grep defaultShow default gateway

Connectivity Tests

ping -c 4 8.8.8.8Ping IP (test raw connectivity)
ping -c 4 example.comPing hostname (tests DNS + connectivity)
traceroute example.comShow network path to host

DNS Lookups

nslookup domain.comBasic name lookup
dig domain.comDetailed DNS query
dig MX gmail.comQuery specific record type

Open Ports & Listening Services

ss -tulnList TCP/UDP listening ports
sudo ss -tulpnInclude process info
nc -vz host portTest if a TCP port is open

Download Files via CLI

wget URLDownload a file
curl -O URLDownload file, keep filename

Interface Statistics

ip -s linkView packet/byte counters
netstat -iLegacy interface stats

Mini Workflows

# Check local IP and default route
ip addr show
ip route | grep default

# Test local DNS resolution
ping -c 4 example.com

# Test port connectivity
nc -vz google.com 443

Tips

  • Test by IP if DNS might be the issue.
  • ss is faster and preferred over netstat on modern systems.
  • Use traceroute to spot network hops and delays.
  • dig +trace domain.com shows full DNS resolution path.
