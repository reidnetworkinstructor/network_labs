Text Viewing, Redirection & Search

Quick reference for working with text files, logs, and CSVs using Linux command-line tools.

Viewing Files

CommandPurposeExample
cat View entire file (small files) cat file.txt
less Page through file with navigation less file.txt (Space=next, /=search, q=quit)
head Show first N lines head -n 10 file.txt
tail Show last N lines; -f follows updates tail -n 20 file.txt â€¢ tail -f logfile

Searching

CommandPurposeExample
grep Search text by pattern grep "ERROR" logfile
grep -i Case-insensitive search grep -i "login" logfile
grep -E Extended regex patterns grep -E "200|404" access.log

Redirection & Append

SymbolPurposeExample
> Write output to file (overwrite) echo "note" > notes.txt
>> Append output to file echo "line2" >> notes.txt

Pipes & Filters

CommandPurposeExample
| (pipe) Send output of one command to another cat file | grep ERROR
tee Write output to screen and file grep "200" access.log | tee ok.log

Data Extraction

CommandPurposeExample
cut Extract specific fields cut -d',' -f2 users.csv
sort Sort lines sort file.txt
uniq Filter duplicate lines uniq names.txt
uniq -c Count occurrences sort names.txt | uniq -c

Mini Workflows

# First 5 lines of log containing 404
grep " 404" access.log | head -n 5

# Count unique status codes
cut -d' ' -f5 access.log | sort | uniq -c | sort -n

# Extract names from CSV where dept=IT
grep ',IT$' users.csv | cut -d',' -f2

# Save non-200 requests to file
grep -v " 200$" access.log > non200.log

Tips

  • Use less for large files to avoid flooding the terminal.
  • Chain commands with | to avoid creating intermediate files.
  • Use grep -E for complex patterns, -i for case-insensitive.
  • For CSVs, cut -d',' -fN is quick; awk is more flexible.
  • Combine sort | uniq -c to produce simple frequency reports.
