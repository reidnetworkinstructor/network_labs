Day 68 Cheat Sheet

Process Management & System Performance

Quick reference for listing, filtering, monitoring, and controlling processes, plus checking system performance metrics.

Listing Processes

CommandPurposeExample
ps auxShow all processesps aux
ps -u usernameShow processes for specific userps -u bob
ps aux | grep nameSearch processes by nameps aux | grep apache

Real-Time Monitoring

topLive process & resource usage
htopImproved, interactive process viewer

Controlling Processes

kill PIDTerminate process gracefully
kill -9 PIDForcefully terminate process
pkill nameKill all processes matching name

CPU Usage & Info

lscpuShow CPU details
uptimeShow uptime & load averages
topLive CPU utilization

Memory Usage

free -hHuman-readable memory usage
topMemory usage per process

Disk Usage

df -hDisk space usage per filesystem
du -sh .Size of current directory

I/O & Performance

iostatCPU & disk I/O stats
iotopTop I/O-using processes

Mini Workflows

# View top 10 CPU-hungry processes
ps aux --sort=-%cpu | head -n 11

# Kill a background process
kill $(pgrep sleep)

# Check load averages & memory
uptime
free -h

Tips

  • Use kill -9 only if normal kill fails.
  • Load average is relative to CPU cores (e.g., 4.00 on 4 cores = fully loaded).
  • htop allows sorting by CPU, memory, or process time with function keys.
  • Combine watch with commands to refresh output every few seconds (watch -n 2 df -h).
