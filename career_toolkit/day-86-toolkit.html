Day 86 Toolkit

One-Page Printable Resume

Copy‑paste blocks and formatting tips to turn your ATS master into a sharp, printable one‑pager.

Pair with: Day 86 Checklist

Workflow: Start from your Day 85 plain‑text resume → trim → light formatting → export to PDF.

1) Recommended One‑Page Layout (Outline)

HEADER
Name | City, ST | Phone | Email | LinkedIn | GitHub/Portfolio

SUMMARY (2–3 sentences)
Certifications + focus + value you bring + target role.

SKILLS & CERTIFICATIONS
Networking | Security | Cloud | Tools  (grouped, concise)
Certifications listed with full names/acronyms.

EXPERIENCE (1–2 roles max)
Title — Company, City, ST (MM/YYYY–MM/YYYY)
• Achievement/result with metric
• Tool/tech + outcome
• Customer or uptime impact

PROJECTS (1–2)
Title — short problem/approach/result | link
• Tools used | measurable learning/outcome

EDUCATION
Degree/Program — School, City, ST (Year)

2) Summary Line Starters

Entry-level IT professional (Network+, AZ‑900) focused on troubleshooting, documentation, and end‑user support. Recent projects include network mapping and log analysis. Seeking Help Desk / IT Support roles to improve resolution time and user experience.
Junior SOC analyst with Security+ (in progress) and hands‑on labs in Splunk and incident documentation. Built a tiny log analyzer to surface failed‑login patterns. Targeting SOC/blue‑team roles where thorough triage and clear runbooks matter.

3) Skills & Certifications (Compact Blocks)

Skills: TCP/IP, DNS, DHCP, VLANs, Wi‑Fi, Windows Admin, Microsoft 365, Azure (AZ‑900), Identity & Access, Ticketing (ServiceNow), PowerShell, Python, Git/GitHub, Splunk (Log Analysis)
Certifications: CompTIA Network+ (N10‑009), Microsoft AZ‑900, CompTIA Security+ (in progress), Splunk Core Certified User

4) Bullet Conversions (From Task → Result)

Weak: Reset passwords and closed tickets.
Strong: Resolved 30+ tickets/week across Windows/M365; cut average resolution time by 25% via clearer triage and KB reuse.
Weak: Worked with Wi‑Fi issues.
Strong: Diagnosed recurring Wi‑Fi drops; documented remediation that reduced repeat incidents by 40% over two months.

5) Project Mini‑Card Templates (1–2 lines each)

Tiny Log Analyzer — Python CLI that summarizes failed logins by IP/user; speeds triage from 15m → <2m. Tools: Python, regex | GitHub: <link>
Network Mapping CLI — Automated local discovery + traceroute write‑up; improved home lab documentation. Tools: Windows CLI, draw.io | GitHub: <link>

6) Visual Polishing Guide (Keep It Light)

• Font: Calibri/Arial/Helvetica. Body 10.5–11.5 pt, Name 14–16 pt.
• Section headers in SMALL CAPS or bold; no color needed.
• 0.5–1" margins; consistent spacing before/after headers.
• Use solid round bullets only; no icons or fancy symbols.
• One column. No text boxes, tables, or images.

7) Space‑Saver Tricks (Without Looking Cramped)

• Collapse older/unrelated roles to one line each.
• Combine Skills & Certifications into one section.
• Limit to 3 bullets/job; keep each bullet to 1–2 lines.
• Use short, scannable phrases; front‑load keywords.

8) File‑Naming & Export

File name: Lastname_Firstname_Resume.pdf
Keep your Day 85 ATS version as ATS_Master.docx
When uploading to portals: use Word (ATS) if requested; send PDF directly to recruiters via email/LinkedIn.

9) Recruiter Reply & Send Script

Subject: Resume — Firstname Lastname
Hi [Name], thanks for reaching out. Attached is my one‑page resume (PDF). Happy to share my ATS version or project links if helpful. Looking forward to next steps.
— Firstname
Hi [Name] — great to connect. Here’s my one‑page resume and portfolio: [link]. My background aligns with [requirement]; happy to elaborate on [project/skill].

10) Quick Review Rubric (Print‑Ready)

SCAN (6–8 sec): Name + role clear? Top skills visible? Any clutter?
CONTENT: Results > tasks? Metrics present? Relevant keywords included?
FORMAT: One page? Clean margins/fonts? Consistent bullets?
LINKS: LinkedIn/GitHub/Portfolio working?
EXPORT: PDF looks identical on phone and desktop?
