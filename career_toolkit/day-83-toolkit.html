3) Recommended Repo Structure

repo-root/
  README.md
  src/              # scripts or code
  docs/             # screenshots, diagrams
  data/             # sample inputs (sanitized)
  .gitignore
  LICENSE           # MIT for personal projects

4) Commit Message Examples

feat: add summary table to log analyzer
fix: handle empty lines gracefully in parser
docs: include screenshot of sample output
refactor: extract parser into separate module
test: add unit test for empty file case
chore: update .gitignore for venv/.env
build: add requirements.txt for quick setup
perf: speed up parsing by using defaultdict
ci: add basic workflow to run tests on push

9) Screenshot & Docs Tips

• Use a clean terminal or UI theme.
• Crop tightly around relevant output.
• Add a short caption in README.
• Store images in /docs and reference relatively.
Markdown example:
![Summary output](docs/summary.png)
Add a one-line explanation below the image describing what the viewer should notice.
