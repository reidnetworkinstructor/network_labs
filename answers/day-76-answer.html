Reference Code – inventory.py

devices = ["router","switch","switch","ap","router","server"]

# Count duplicates
counts = {}
for d in devices:
    counts[d] = counts.get(d, 0) + 1

print("Original:", devices)
print("Counts:")
for k, v in counts.items():
    print(f"  {k} = {v}")

# Deduplicate while keeping order
unique = []
for d in devices:
    if d not in unique:
        unique.append(d)

# Ask user for a device name to add/remove
action_device = input("Enter a device to add/remove: ").strip().lower()

if action_device in unique:
    unique.remove(action_device)
    print(f"Removed one '{action_device}'")
else:
    unique.append(action_device)
    print(f"Added '{action_device}'")

# Sort and print with line numbers
unique.sort()
print("Final inventory:")
for i, dev in enumerate(unique, start=1):
    print(f"{i}. {dev}")

Reference Code – defaults.py

# Constant settings in tuples
dns = ("8.8.8.8", "1.1.1.1")
ntp = ("time.google.com", "time.cloudflare.com")

primary_dns, secondary_dns = dns
primary_ntp, secondary_ntp = ntp

print(f"Primary DNS: {primary_dns}, Secondary DNS: {secondary_dns}")
print(f"Primary NTP: {primary_ntp}, Secondary NTP: {secondary_ntp}")

# Attempt to change tuple element (commented to avoid error)
# dns[0] = "9.9.9.9"  # ❌ TypeError: 'tuple' object does not support item assignment
print("Note: Tuples are immutable – you cannot change their elements.")

# Record tuple for a user
user = ("jkim", "Jordan Kim", "student")
username, full_name, role = user
print(f"User '{username}' ({full_name}) has role '{role}'.")

Why This Works

  • inventory.py: Uses a dictionary to count occurrences, a list to store deduplicated values, and enumerate to number output.
  • defaults.py: Demonstrates tuple unpacking and immutability, making it clear why tuples are good for constants.

Common Mistakes & Fixes

  • Forgetting to strip/normalize user input → could fail to match existing items in unique list.
  • Sorting before deduplicating → changes intended order before count check.
  • Attempting to modify a tuple element → raises TypeError.

Rubric (Pass/Complete)

  • inventory.py correctly prints original list, counts, handles add/remove, sorts final list, and numbers items.
  • defaults.py prints unpacked tuple values and explains immutability.
  • Both scripts run without errors and match the required structure.
