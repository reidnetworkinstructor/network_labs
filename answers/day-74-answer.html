Reference Code for triage.py

# Gather inputs
severity = int(input("Severity (1-5): "))
category = input("Category (network/password/malware): ").strip().lower()
hours_open = int(input("Hours open: "))
vip_user = input("VIP user? (y/n): ").strip().lower()
vip = (vip_user == "y")

# Decision logic
if severity >= 4 or (vip and severity >= 3):
    action = "Page on-call now"
    if category == "network":
        action += " and create network incident bridge"
elif category == "malware":
    action = "Isolate host and escalate to security"
elif category == "network" and hours_open >= 2:
    action = "Create network incident bridge"
elif category == "password":
    action = "Guide reset (verify identity)"
else:
    action = "Queue for next available technician"

# Output
print("\n=== Ticket Summary ===")
print(f"Severity: {severity}")
print(f"Category: {category}")
print(f"Hours Open: {hours_open}")
print(f"VIP: {vip}")
print(f"\nRecommended Action: {action}")

Why This Works

  • Uses comparison operators to evaluate severity and hours open.
  • Combines conditions with or and and for VIP escalation logic.
  • Normalizes input with .strip().lower() to prevent mismatches from extra spaces or casing.
  • Orders branches so that critical/high-priority conditions are checked before less urgent cases.

Common Mistakes & Fixes

  • Logic order issues: Putting a general else before more specific cases will skip them — fix by ordering from most specific/urgent to least.
  • TypeError: Forgetting to convert input() values to int before numeric comparisons.
  • String matching errors: Not normalizing case, causing “Network” to fail equality checks.

Rubric (Pass/Complete)

  • Prompts for severity, category, hours_open, and VIP.
  • Handles all major categories (network/password/malware) plus a default route.
  • Correctly escalates based on severity, VIP status, and time open.
  • Output matches the summary format shown in the lab instructions.
