Reference Code for badge.py

# Gather inputs
first_name = input("First name: ").title()
last_name = input("Last name: ").title()
city = input("City: ").title()
role = input("Role: ").title()
fav_skill = input("Favorite Hoodie Skill: ").title()
years_exp = int(input("Years of IT Experience: "))

# Derived values
initials = first_name[0].upper() + last_name[0].upper()
tagline = f"{first_name} builds {fav_skill} skills in {city}."
user_id = f"{initials}{str(len(city)).zfill(2)}"
username = (first_name[0] + last_name).lower()

# Output badge
print("="*30)
print(" Hoodie Launchpad – Onboarding ")
print("="*30)
print(f"Name: {first_name} {last_name}")
print(f"City: {city}")
print(f"Role: {role}")
print(f"Favorite Skill: {fav_skill}")
print(f"Years of IT Experience: {years_exp}")
print(f"Initials: {initials}")
print(f"ID: {user_id}")
print(f"Username: {username}")
print(f"Tagline: {tagline}")

Why This Works

  • input() reads text from the user; .title() ensures consistent casing for names and city.
  • F-strings embed variables inline without clunky concatenation.
  • Derived values like initials, ID, and username showcase string indexing, length functions, and method chaining.
  • Multi-line, formatted output improves readability and user experience.

Common Mistakes & Fixes

  • ValueError: Trying to convert non-numeric years to int — fix by validating with isdigit() before conversion.
  • IndexError: If first/last name is left blank — ensure the user actually enters a value before indexing.
  • Forgetting .title() on names — leads to inconsistent casing in output.

Rubric (Pass/Complete)

  • Program prompts for all required inputs.
  • Names, city, role, and skill are normalized to title case.
  • Derived values (initials, tagline) computed and displayed correctly.
  • Output matches the example structure.
