Step 1: Set Up GNS3

  • Download and install GNS3 (free version). Use the basic local setup (no license required).
  • Install the GNS3 VM via VirtualBox or VMware as prompted.
  • Create a new project in GNS3: HoodieRoutingLab.
  • Drag the following into the topology:
    • 2 Routers (e.g., use vIOS images)
    • 1 Switch
    • 1 PC (VPCS)
  • Connect:
    • Router1 ‚Üî Router2: Serial link 10.0.0.0/30
    • Router1 ‚Üî Switch: Ethernet 192.168.1.0/24

Step 2: Configure OSPF Routing

  • Start all nodes (green play button).
  • On Router1 Console:
    • enable
configure terminal
interface s0/0
ip address 10.0.0.1 255.255.255.252
no shutdown
exit
router ospf 1
network 10.0.0.0 0.0.0.3 area 0
exit
  • On Router2 Console:
    • enable
configure terminal
interface s0/0
ip address 10.0.0.2 255.255.255.252
no shutdown
exit
router ospf 1
network 10.0.0.0 0.0.0.3 area 0
exit
  • On VPCS PC:
    • ip 192.168.1.10/24 192.168.1.1
  • Test: From PC run ping 10.0.0.2 ‚Äî should work!
Log: ‚ÄúOSPF configured‚Äîdynamic routing between 192.168.1.0/24 and 10.0.0.0/30.‚Äù

Step 3: Simulate Cable and PoE Issues

  • Cabling Fault: In GNS3, suspend the link between Switch and PC.
  • On Windows Command Prompt:
    • ping 192.168.1.1
  • Result: Ping should fail (mimics Layer 1 cable fault).
  • Open Resource Monitor > Network tab ‚Äî view dropped packets or low throughput.
  • Resume link in GNS3 ‚Üí Ping should work again.
  • Log: ‚ÄúSuspended link mimicked crosstalk‚Äîreconnected, Layer 1 fixed.‚Äù
  • PoE Issue: Assume switch is powering WAP via PoE (802.3at).
  • In Notepad, simulate PoE failure: ‚ÄúWAP at 192.168.1.20 unresponsive‚ÄîPoE failure suspected.‚Äù
  • On Windows:
    • ping 192.168.1.20
netstat -an
  • Assume PoE restored: ‚ÄúWAP online after enabling PoE on port.‚Äù
  • Log: ‚ÄúPoE restored‚ÄîWAP at 192.168.1.20 up, Layer 2 powered.‚Äù
