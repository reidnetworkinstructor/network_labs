Map a Packet’s Journey Through the OSI Model

Visualize how data flows from your browser to a server across all 7 OSI layers

Objective

Trace the path of a packet as it moves through the 7 layers of the OSI model when accessing a website like google.com.

Step 1: Scenario Setup

Imagine this situation: You open your browser and type in google.com. What happens behind the scenes as that request moves through the network?

Step 2: Trace the Packet Through the OSI Layers

  • Layer 7 (Application): Your browser (e.g., Chrome) creates an HTTP request to access google.com.
  • Layer 6 (Presentation): The request is formatted and encrypted using HTTPS for secure transmission.
  • Layer 5 (Session): A session is established between your browser and Google’s server to manage communication.
  • Layer 4 (Transport): TCP breaks the request into packets and ensures reliable delivery.
  • Layer 3 (Network): Your router assigns a source IP address and routes the packets toward Google’s server IP.
  • Layer 2 (Data Link): Each packet is framed and sent over your local network (Wi-Fi or Ethernet).
  • Layer 1 (Physical): Data travels as electrical signals, light, or radio waves across the physical medium.

Step 3: Visualize the Journey

Create a diagram that shows:

  • Your Computer → Router → Internet → Google’s Server
  • Label the function of each OSI layer along the journey.
  • Use arrows or boxes to represent each step.
