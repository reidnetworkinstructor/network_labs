Day 67 Lab

Network & Connectivity Commands

Objective: Use Linux networking tools to view interface details, test connectivity, resolve DNS names, trace routes, and check open ports/services.

ðŸ›  Tools Needed

  • Linux system (VM, WSL, or cloud instance)
  • Terminal access
  • Internet connectivity for external tests

Step 1: View IP Address & Interfaces

# Modern tool
ip addr show

# Shorter form (only IPv4/IPv6 addresses)
ip -brief addr

# Legacy ifconfig (if installed)
ifconfig

Step 2: Check Routing Table & Default Gateway

# Show routing table
ip route show

# Show default route only
ip route | grep default

Step 3: Test Basic Connectivity (ping)

# Ping external host (Ctrl+C to stop)
ping -c 4 8.8.8.8

# Ping by hostname
ping -c 4 google.com

Step 4: Trace the Path to a Host

# Install traceroute if missing (Debian/Ubuntu example)
# sudo apt install traceroute
traceroute google.com

Step 5: DNS Lookups

# Simple lookup
nslookup openai.com

# Detailed (if dig installed)
dig openai.com

# Query specific record type
dig MX gmail.com

Step 6: View Current Network Connections

# List listening ports (netstat variant)
ss -tuln

# With process info (sudo may be needed)
sudo ss -tulpn

Step 7: Test Port Connectivity

# Using nc (netcat) to test TCP connection
nc -vz google.com 80

# Test multiple ports
nc -vz google.com 80 443

Step 8: Download a File via CLI (curl/wget)

# wget example
wget https://example.com

# curl example
curl -O https://example.com/index.html

Step 9: Display Interface Statistics

# Summary statistics
ip -s link

# Old style
netstat -i
