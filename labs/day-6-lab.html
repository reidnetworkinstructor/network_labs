Day 6 Lab

Investigate DNS and DHCP with Command Prompt

Objective: Use Windows Command Prompt to explore how DNS resolves domain names and how DHCP assigns your IP address — connecting these protocols to your daily internet use.

Tools Needed: Command Prompt (built-in on Windows — search for “cmd” in the Start menu).

Step 1: Check Your DHCP Configuration

  1. Open Command Prompt: Press Win + R, type cmd, and press Enter.
  2. Type ipconfig /all and press Enter to view your network configuration.
  3. Look for:
    • IPv4 Address (e.g., 192.168.1.100).
    • DHCP Server (likely your router, e.g., 192.168.1.1).
    • DHCP Enabled (should say “Yes” if your IP is assigned automatically).
  4. Write these details in your notes for reference.

Step 2: Test DNS Resolution

  1. In Command Prompt, type nslookup google.com and press Enter.
  2. Look for the Address field (e.g., 142.250.190.78).
  3. Repeat with another domain: nslookup facebook.com.
  4. Record both IP addresses alongside your DHCP notes.

Step 3: Reflect

Write a short paragraph: How does DHCP simplify network setup? What did you notice about DNS resolution?

Hint: DHCP automatically assigns IPs, eliminating manual setup. DNS quickly translates domain names to IP addresses so browsers can locate websites.
