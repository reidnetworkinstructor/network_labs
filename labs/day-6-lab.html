Investigate DNS and DHCP with Command Prompt

Explore how your computer automatically gets an IP and resolves domain names.

Objective

Use Command Prompt to explore how DNS resolves domain names and how DHCP assigns your IP address, connecting these protocols to your daily internet use.

Tool Needed

Command Prompt (built-in on Windows—search for “cmd” in the Start menu).

Step 1: Check Your DHCP Configuration

  1. Open Command Prompt: Press Win + R, type cmd, and press Enter.
  2. Type ipconfig /all and press Enter to view your network configuration.
  3. Look for:
    • Your IPv4 address (e.g., 192.168.1.100).
    • DHCP Server (likely your router, e.g., 192.168.1.1).
    • DHCP Enabled (should say “Yes” if your IP is assigned automatically).
  4. Note these details in a text file or notebook.

Step 2: Test DNS Resolution

  1. Type nslookup google.com and press Enter to see how DNS resolves the domain name to an IP address.
  2. Look for the “Address” field in the response (e.g., 142.250.190.78).
  3. Try another domain, like nslookup facebook.com, and note the IP address.
  4. Note these results alongside your DHCP findings.

Step 3: Reflect

Write a short paragraph: How does DHCP simplify your network setup? What did you notice about the DNS resolution process?

Hint: DHCP automates IP assignment, saving manual effort; DNS quickly translates names to IPs, enabling web browsing.
