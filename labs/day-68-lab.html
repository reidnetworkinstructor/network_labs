Day 68 Lab

Process Management & System Performance

Objective: Learn how to monitor running processes, analyze system performance, and control processes using Linux commands and tools.

ðŸ›  Tools Needed

  • Linux system (VM, WSL, or physical machine)
  • Terminal access

Step 1: View Running Processes

# Show all processes
ps aux

# Show processes for current user
ps -u $(whoami)

Step 2: Filter Processes

# Find a process by name
ps aux | grep firefox

# Exclude the grep process itself
ps aux | grep firefox | grep -v grep

Step 3: Real-Time Process Monitoring

# Launch top (press q to quit)
top

# Launch htop if installed (color, easier navigation)
htop

Step 4: Kill a Process

# Kill by PID
kill 1234

# Force kill
kill -9 1234

Step 5: Check CPU Usage

# View CPU info
lscpu

# Check load averages and CPU utilization
uptime
top

Step 6: Check Memory Usage

# Human-readable memory summary
free -h

# Memory usage in top
top

Step 7: Check Disk Usage

# Human-readable disk space
df -h

# Disk usage of current directory
du -sh .

Step 8: Check I/O and Performance

# Install iostat (Debian/Ubuntu example)
# sudo apt install sysstat

# Show I/O stats
iostat

# Show process I/O usage
iotop
