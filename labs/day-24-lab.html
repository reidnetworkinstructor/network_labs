Automate & Monitor with Ansible and PRTG

Objective: Use Ansible to push a small, repeatable ‚Äúconfig‚Äù (simulated) and use PRTG to monitor simple network health (ping/HTTP). No SNMP required.

Step 2 ‚Äî Your First Idempotent Playbook

This playbook ‚Äúpretends‚Äù to push network config by writing a versioned note to a file. Running it twice should show ok (no change).

cat > vars.yml << 'EOF'
vlan_id: "10"
snmp_trap_host: "192.168.1.50"
config_file: "./config_push.txt"
EOF

cat > playbook.yml << 'EOF'
- name: HL Day24 - Simulated Config Push
  hosts: local
  gather_facts: false
  vars_files:
    - vars.yml
  tasks:
    - name: Ensure config file exists
      ansible.builtin.file:
        path: "{{ config_file }}"
        state: touch

    - name: Write VLAN note (idempotent)
      ansible.builtin.lineinfile:
        path: "{{ config_file }}"
        regexp: "^VLAN:"
        line: "VLAN: {{ vlan_id }}"
        create: yes

    - name: Write SNMP trap host note (idempotent)
      ansible.builtin.lineinfile:
        path: "{{ config_file }}"
        regexp: "^SNMP trap host:"
        line: "SNMP trap host: {{ snmp_trap_host }}"
        create: yes

    - name: Append timestamped audit entry
      ansible.builtin.lineinfile:
        path: "{{ config_file }}"
        line: "Applied at {{ lookup('pipe','date -u +%F_%T_UTC') }}"
        insertafter: EOF
EOF

ansible-playbook -i inventory.ini playbook.yml

Run it again; the first two tasks should report ok (unchanged), proving idempotency:

ansible-playbook -i inventory.ini playbook.yml
cat config_push.txt

Step 3 ‚Äî Parameterize a Change

Edit vars.yml to use vlan_id: "20" and re-run the playbook. The VLAN line should change while the SNMP line remains ok.

Step 4 ‚Äî PRTG: Quick Monitoring (No SNMP)

  1. Open PRTG web interface (Start Menu ‚Üí PRTG Network Monitor).
  2. In the PRTG tree, add a Device:
    • Name: HL-Gateway
    • IPv4/Hostname: your default gateway IP (e.g., 192.168.1.1)
  3. Add sensors to that device:
    • Ping (default sensor) ‚Äî confirms basic reachability.
    • HTTP ‚Äî monitor a URL like https://www.microsoft.com.
  4. Let it collect a few minutes of data.

Step 5 ‚Äî Simulate an Issue & Observe

  • Temporarily disconnect Wi-Fi or unplug Ethernet for ~20 seconds.
  • Re-connect and watch PRTG:
    • Ping should turn red (down) during the outage, then green (up) again.
    • HTTP may show errors during the outage.
