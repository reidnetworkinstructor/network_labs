Day 24 Premium Lab:

Automate and Monitor with Ansible and PRTG

Step 1: Set Up Tools

  • Ansible:
    • Install Windows Subsystem for Linux (WSL):
      • wsl --install
    • Launch WSL and install Ansible:
      • sudo apt update && sudo apt install ansible
  • PRTG:
    • Download PRTG from paessler.com (100 sensors free).
    • Install and access via localhost:8080 on your browser.

Step 2: Simulate Automation with Ansible

  • In WSL, create a playbook file:
    • nano playbook.yml 
    - hosts: localhost
  tasks:
    - name: Simulate VLAN config
      debug:
        msg: "Configured VLAN 10 on switch"

    - name: Simulate SNMP trap setup
      debug:
        msg: "SNMP trap host set to 192.168.1.50"
  • Run the playbook:
    • ansible-playbook playbook.yml
  • Review output ‚Äî should show both tasks completed with debug messages.

Step 3: Monitor with PRTG

  • In PRTG, add your local PC as a device.
  • Auto-discover sensors or manually add a new SNMP Traffic Sensor to monitor your network adapter.
  • Simulate a network issue:
    • Disable and re-enable your Ethernet or Wi-Fi adapter.
    • Watch PRTG register link down and traffic spikes.
  • View traffic charts, latency, and alerts in the PRTG dashboard.
