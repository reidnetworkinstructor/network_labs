IoT & Cloud Security Audit

Objective: Simulate a security audit of a cloud-based storage bucket and a vulnerable IoT device. Identify exposures and document mitigation strategies.

Scenario:

You are hired to audit a small company that uses smart security cameras and stores footage in cloud buckets. Two major risks have been flagged:

  • Unsecured S3 bucket storing camera footage
  • Unprotected IoT camera feed accessible online

Step 1: Simulate Cloud Bucket Exposure

  • Go to the S3 Bucket Scanner
  • Search for public buckets using company-style keywords (e.g., companyname-backups, cctv-footage)
  • Document if files are publicly listed, downloadable, or misconfigured

Step 2: Simulate IoT Camera Exploitation

  • Visit Insecam â€“ US feed
  • Find a live public camera feed that shouldn't be exposed
  • Note the IP, location, and whether the admin interface is reachable
  • Try appending :80 or :8080 to the IP in your browser
  • Does a login prompt appear? Any default credentials exposed?

Step 3: Risk Documentation

  • For both systems, write down:
    • What is exposed?
    • Why is it a risk?
    • What should be done to fix it?
  • Save your notes as iot-cloud-audit.txt

Step 4: Mitigation Planning

  • List at least 3 security controls for each of the following:
    • Cloud Buckets: (e.g., enable bucket policies, disable public access)
    • IoT Devices: (e.g., change default credentials, VLAN segmentation, firmware updates)

Optional Bonus:

  • Use nmap -sV [IP] to scan the IoT device from Insecam if reachable
  • Create a short audit presentation slide deck for your security team
