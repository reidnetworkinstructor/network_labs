Subnetting with CIDR - Diagramming Networks

Use diagrams.net to visually break down a Class C network into multiple subnets.

Objective

Use diagrams.net to create a visual subnet diagram for a network divided using CIDR notation, mastering Network+ subnetting skills.

Tools Needed

  • Diagrams.net (free, web-based)
  • Paper/pen or Notepad for calculations
  • Optional: Windows Calculator in Programmer mode

Step 1: Plan Your Network

Start with a Class C network: 192.168.1.0/24 (256 addresses).

Goal: Divide it into 3 subnets:

  • IT – needs ~120 hosts
  • Sales – needs ~60 hosts
  • Guest – needs ~30 hosts

Step 2: Calculate Subnets with CIDR

  • IT: /25 → 192.168.1.0 - 192.168.1.127 (126 usable)
  • Sales: /26 → 192.168.1.128 - 192.168.1.191 (62 usable)
  • Guest: /27 → 192.168.1.192 - 192.168.1.223 (30 usable)

Optional: Use Windows Calculator (Programmer mode) to verify block sizes and masks.

Step 3: Create Diagram in Diagrams.net

  1. Open diagrams.net.
  2. Use network shapes to create:
    • Cloud labeled “Internet”
    • Router labeled “192.168.1.1/24”
    • Three subnet blocks:
      • IT: 192.168.1.0/25
      • Sales: 192.168.1.128/26
      • Guest: 192.168.1.192/27
  3. Add IP ranges, examples (e.g., IT Server at .10), and icons.
  4. Save as PNG or PDF.

Step 4: Reflect

Write a short paragraph: How did diagramming help you understand CIDR subnetting?

Hint: Visualizing /25, /26, and /27 makes Layer 3 subnetting easier to grasp!
