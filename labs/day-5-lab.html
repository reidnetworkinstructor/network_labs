Day 5 Lab

Subnetting with CIDR — Diagramming Networks

Objective: Use diagrams.net to create a visual subnet diagram for a network divided using CIDR notation, reinforcing Network+ subnetting skills.

Tools Needed:

  • Diagrams.net (free, browser-based)
  • Paper/pen or Notepad for manual calculations
  • Optional: Windows Calculator in Programmer mode

Step 1: Plan Your Network

Starting network: 192.168.1.0/24 (256 total addresses).

Goal: Divide it into 3 subnets to meet department needs:

  • IT — ~120 hosts
  • Sales — ~60 hosts
  • Guest — ~30 hosts

Step 2: Calculate Subnets with CIDR

  • IT: /25 → 192.168.1.0 – 192.168.1.127 (126 usable hosts)
  • Sales: /26 → 192.168.1.128 – 192.168.1.191 (62 usable hosts)
  • Guest: /27 → 192.168.1.192 – 192.168.1.223 (30 usable hosts)

Optional: Use Windows Calculator (Programmer mode) to verify block sizes and subnet masks.

Step 3: Create Diagram in diagrams.net

  1. Go to diagrams.net and start a new blank diagram.
  2. Add a cloud labeled Internet.
  3. Add a router labeled 192.168.1.1/24.
  4. Create three subnet blocks:
    • IT: 192.168.1.0/25
    • Sales: 192.168.1.128/26
    • Guest: 192.168.1.192/27
  5. Include IP ranges, sample devices (e.g., IT Server at .10), and icons for clarity.
  6. Save the diagram as PNG or PDF for your portfolio.

Step 4: Reflect

Write a short paragraph on how visualizing CIDR subnets helped your understanding of IP addressing and block allocation.

Hint: Seeing /25, /26, and /27 subnets visually makes it easier to grasp how address space is divided and used efficiently.
