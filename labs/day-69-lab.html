Day 69 Lab

Disk & File System Management

Objective: Learn how to view, manage, and troubleshoot disks and file systems in Linux.

ðŸ›  Tools Needed

  • Linux system (VM, WSL, or physical machine)
  • Terminal access
  • lsblk, df, du, mount, umount installed

Step 1: View Attached Storage Devices

# Show block devices
lsblk

# Show with file system info
lsblk -f

# Detailed disk usage info
sudo fdisk -l

Step 2: Check File System Type

# List mounted file systems
mount | column -t

# Show file system types
df -T

Step 3: Check Disk Space

# Human-readable disk space
df -h

# Space usage by directory
du -sh /var/log

# Top 10 largest directories/files
du -ah / | sort -rh | head -n 10

Step 4: Mount a New File System

# Create a mount point
sudo mkdir /mnt/mydisk

# Mount a device (replace sdb1 with your device name)
sudo mount /dev/sdb1 /mnt/mydisk

# Verify mount
mount | grep mydisk

Step 5: Unmount a File System

# Unmount by mount point
sudo umount /mnt/mydisk

# Or by device name
sudo umount /dev/sdb1

Step 6: Check & Repair File System

# Check for errors (unmounted device only)
sudo fsck /dev/sdb1

Step 7: Create a New File System

# WARNING: This will erase the disk partition
sudo mkfs.ext4 /dev/sdb1
