Day 8 Lab

Analyze Network Traffic with Wireshark

Objective: Use Wireshark to capture and analyze HTTP, HTTPS, and DNS traffic. Observe how application-layer protocols use specific ports and understand the role of ports in communication.

Tools Needed: Wireshark (free network protocol analyzer).

Step 1: Install and Set Up Wireshark

  1. Download and install Wireshark on your computer (install Npcap if prompted).
  2. Open Wireshark and select your active network interface (e.g., Wi-Fi or Ethernet).
  3. Click the green shark fin icon to begin capturing packets.

Step 2: Generate and Capture HTTP Traffic

  1. In your browser, visit a non-secure site using http:// (e.g., http://example.com).
  2. Stop the capture once the page loads (red square icon).
  3. In the filter bar, type tcp.port == 80 and press Enter.
  4. Look for packets with “HTTP” in the Protocol column and inspect their source/destination ports.
  5. Note example: “HTTP traffic on port 80, source port [random], destination port 80.”

Step 3: Generate and Capture HTTPS and DNS Traffic

  1. Start a new capture.
  2. Visit https://google.com in your browser.
  3. Stop the capture.
  4. Filter for HTTPS: tcp.port == 443 — look for “TLS” packets.
  5. Filter for DNS: udp.port == 53 — look for “DNS” query and response packets.
  6. Note example: “HTTPS traffic on port 443, DNS query for google.com on port 53.”

Step 4: Reflect

Write a short paragraph: How did Wireshark help you understand the role of ports in communication? What did you learn about protocol-port relationships?

Hint: HTTP uses port 80, HTTPS uses port 443, DNS uses port 53 — ports guide traffic to the correct service.
