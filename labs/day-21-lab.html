Day 21 Premium Lab:

Set Up Your First Azure Virtual Network (VNet)

Step 1: Set Up Your Free Azure Account

Visit azure.microsoft.com/free and sign up. You'll need an email and credit card (no charges for free tier).

Log in to the Azure Portal.

Log: “Azure account created—ready to build a cloud network!”

Step 2: Create a Virtual Network (VNet)

In the Azure Portal:

  1. Click Create a Resource > search for Virtual Network > select and click Create.
  2. Configure:
    • Subscription: Free Trial
    • Resource Group: Create new → HoodieCloud
    • Name: HoodieVNet
    • Region: Closest region (e.g., East US)
    • Address Space: 10.0.0.0/16
    • Subnet: Name: PublicSubnet, Address Range: 10.0.1.0/24
  3. Click Review + Create → Create (takes ~1 min).
  4. Check All Resources to confirm HoodieVNet is listed.
Log: “Created VNet 10.0.0.0/16 with PublicSubnet 10.0.1.0/24—IaaS in action!”

Step 3: Explore and Simulate

  • View the VNet’s Subnets tab.
  • Note that the subnet range is 10.0.1.0 - 10.0.1.255.
  • Imagine a web server at 10.0.1.10.
  • Simulate a security group by writing in Notepad: Permit TCP 443 (HTTPS) to 10.0.1.0/24, deny 23 (Telnet).

📷 Take a screenshot of the VNet details page (address space + subnet shown).

Note: “VNet set, HTTPS allowed—cloud network secure and functional.”

Step 4: Reflect

Write a short paragraph:

How does this VNet reflect cloud networking?

Hint: You created a 10.0.0.0/16 VNet with a /24 subnet, mimicking real IaaS cloud networks—just like you'd see in Network+ objectives.
