Day 11 Lab

Crimp and Test an Ethernet Cable

Objective

Build a straight-through Ethernet cable and test it in both the real world and simulation to understand how Physical Layer (Layer 1) connections work.

Tools Needed

  • UTP cable (Cat 5e or Cat 6)
  • 2 × RJ45 connectors
  • Crimping tool
  • Wire stripper
  • Optional: Cable tester
  • Optional: Cisco Packet Tracer

Step 1: Prep the Cable

Cut 1–2 meters of UTP. Strip ~2 inches of the outer jacket to expose the four twisted pairs. Untwist and arrange the conductors to the T568B order:

Orange-White, Orange, Green-White, Blue, Blue-White, Green, Brown-White, Brown

Line the wires flat, trim evenly to ~0.5", and slide into the RJ45 with the clip facing down. Orange-White should be Pin 1 (left).

Step 2: Crimp It

Seat the cable fully so each wire reaches the end of the connector. Verify the order is still correct, then crimp firmly.

Repeat for the other end using the same T568B pinout to make a straight-through cable.

Check the jacket is caught by the strain-relief tab and that all copper pins are fully pressed and visible.

Step 3: Physical Test

Plug one end into a computer and the other into a switch or router. Look for link/activity lights. Try browsing or pinging your default gateway.

If you have a tester, connect both ends—lights should run 1→8 in order with no splits or reversals.

Example: “Crimped Cat 6 cable — link established and internet reachable.”

Step 4: Simulate It (Optional)

  • In Packet Tracer, add two PCs and a switch.
  • Use the copper straight-through tool to connect each PC to the switch.
  • Assign IPs: PC0 = 192.168.1.10, PC1 = 192.168.1.11.
  • From PC0, ping PC1. You should receive replies.
Result: “Simulated cable working — ping reply received.”

Step 5: Reflect

  • How does a properly crimped cable enable Layer 1 communication?
  • Why use the same T568B standard on both ends for a straight-through cable?

Hint: Standard pinouts ensure correct transmit/receive pair alignment between devices.
