Using Basic Search in Splunk

Objective: Practice core Search Processing Language (SPL) skills: keyword queries, Boolean logic, field/value filters, wildcards, time range control, and simple result shaping with fields, table, dedup, and sort.

Step 1: Open Search & Set Time

  1. Sign in ‚Üí Apps ‚ñ∏ Search & Reporting.
  2. Top-right time range: choose Last 60 minutes.
  3. Ensure Search mode is Smart (next to the time picker).

Step 2: Keyword & Phrase Search

We‚Äôll use built-in data so you always have results.

  • Run: index=_internal error
  • Try a phrase search (quotes look for exact phrases):
    index=_internal "rest handler"
  • Note: Searches are case-insensitive by default for keywords.

Step 3: Boolean Logic

Use AND, OR, NOT and parentheses to control logic.

  • Errors or warnings:
    index=_internal (error OR warn)
  • Errors but not UI-related:
    index=_internal error NOT ui
  • Group logic:
    index=_internal (error OR fail) (http OR tcp)

Step 4: Field/Value Filters

Filtering by fields is faster and cleaner than keyword-only.

  • By sourcetype:
    index=_internal sourcetype=splunkd
  • By source path (wildcard):
    index=_internal source="*metrics.log"
  • Combine:
    index=_internal sourcetype=splunkd (error OR warn)
  • Host-specific (example):
    index=_internal host=*splunk*

Step 5: Wildcards & Quoting

  • Wildcard * matches any characters at that position.
    Example: source="*scheduler.log"
  • Use quotes when values include spaces or special chars.
    Example: sourcetype="splunkd_ui_access"

Step 6: Time Control in the Search Bar

You can override the picker using earliest/latest in the search.

  • Last 2 hours, rounded to the hour:
    index=_internal earliest=-2h@h latest=@h
  • Yesterday only:
    index=_internal earliest=@d-1d latest=@d

Step 7: Shape the Results (Fields, Table, Dedup, Sort)

Now turn noisy events into readable lists.

  • Show only specific fields:
    index=_internal sourcetype=splunkd error | fields _time host source sourcetype
  • Tabular view for quick skimming:
    index=_internal sourcetype=splunkd warn | table _time host source log_level
  • Unique values by field (no repeats):
    index=_internal sourcetype=splunkd | dedup host | table _time host source
  • Sort by most recent first:
    index=_internal error | sort - _time

Step 8: Quick Checks That Speed You Up

  • Fields sidebar: Expand Interesting Fields; click a value to filter.
  • Event sampling: Set to 1:10 while exploring to speed up, then turn off to finalize.
  • Search Inspector: (dropdown next to the job menu) to see performance details.

Step 9: Save a Reusable Search (Report)

  1. Run a clean filter you like, e.g.:
    index=_internal sourcetype=splunkd (error OR warn) | table _time host source log_level
  2. Click Save As ‚ñ∏ Report ‚Üí name it Day52: Recent Splunkd Errors/Warnings.
  3. Add a short description and save. (We‚Äôll use reports again when you build alerts.)
