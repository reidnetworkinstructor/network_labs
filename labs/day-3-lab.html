Explore IP Configuration with Command Prompt

Investigate IP details, ping public and local devices, and explore ARP cache with Windows CMD.

Objective

Use Command Prompt to explore your computer’s IP configuration, test connectivity, and examine the ARP cache, connecting IP addressing to protocols like DHCP and ARP.

Tool Needed

Command Prompt (built-in on Windows—search for “cmd” in the Start menu).

Step 1: Check Your IP Configuration

  • Open CMD: Press Win + R, type cmd, press Enter.
  • Type ipconfig /all and press Enter.
  • Note these values:
    • IPv4 Address (e.g., 192.168.1.100)
    • Subnet Mask (e.g., 255.255.255.0)
    • Default Gateway (e.g., 192.168.1.1)
    • DHCP Server (usually your router)

Step 2: Test Connectivity with Ping

  • Ping Google DNS: ping 8.8.8.8
  • Did it reply? What was the average round-trip time?
  • Now ping your Default Gateway: ping 192.168.1.1
  • Why is the local ping likely faster? (Hint: LAN vs. WAN)

Step 3: Examine the ARP Cache

  • Type arp -a and press Enter.
  • Note at least two entries:
    • e.g., 192.168.1.1 → 00-14-22-01-23-45
    • What other devices show up?

Reflect

Write a paragraph: How does ARP help with LAN communication? Why is the gateway’s MAC in the cache?

Hint: ARP translates IPs to MACs. Your computer needs the MAC of your gateway to send outbound traffic.
