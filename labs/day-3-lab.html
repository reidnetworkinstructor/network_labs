Explore IP Configuration with Command Prompt

Objective: Use Windows Command Prompt to inspect your computer’s IP configuration, test connectivity with ping, and examine the ARP cache—connecting IP addressing to protocols like DHCP and ARP.

Tools Needed: Command Prompt (built-in on Windows — press Win + R, type cmd, press Enter).

Step 1: Check Your IP Configuration

  • Open CMD: Win + R → cmd → Enter.
  • Run: ipconfig /all
  • Record the following:
    • IPv4 Address (e.g., 192.168.1.100)
    • Subnet Mask (e.g., 255.255.255.0)
    • Default Gateway (e.g., 192.168.1.1)
    • DHCP Server (often your router)

Step 2: Test Connectivity with Ping

  • Ping Google DNS: ping 8.8.8.8
  • Did it reply? Note the Average round‑trip time.
  • Ping your default gateway: ping 192.168.1.1 (adjust if yours is different)
  • Which is faster and why? Hint: LAN latency vs WAN/Internet latency.

Step 3: Examine the ARP Cache

  • Run: arp -a
  • Note at least two entries, e.g.:
    • 192.168.1.1 → 00-14-22-01-23-45 (your gateway)
    • Another local device’s IP → MAC

Reflect

Write a short paragraph: How does ARP enable LAN communication? Why is the gateway’s MAC in your ARP cache?

Hint: ARP resolves IP → MAC so frames can be delivered on the local network. Your PC needs the gateway’s MAC to send traffic off‑subnet.
