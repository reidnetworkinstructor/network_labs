Day 14 Lab

Network Services and Applications in Action

Objective: Explore common network services like DHCP, DNS, and NTP using built-in tools, and build familiarity with how your computer communicates on the network.

🔓 Explore Services with Command Prompt

🛠 Tools Needed:

  • Windows PC
  • Command Prompt
  • Internet access

Step 1: Check DHCP

  • Open Command Prompt (press Win + R, type cmd and hit Enter).
  • Type ipconfig /all
  • Look for “DHCP Server” (usually your router) and your assigned IP.
  • Example: “DHCP from router 192.168.1.1, IP assigned—DORA process successful.”

Step 2: Test DNS

  • Type nslookup google.com — it should return an IP and show your DNS server.
  • Try nslookup facebook.com for comparison.
  • Example: “DNS resolves fast—using 8.8.8.8 on port 53.”

Step 3: Check NTP

  • Type w32tm /query /status
  • Check the “Source” (e.g., time.windows.com) and last sync time.
  • Example: “NTP synced at 10:05 AM—clock accuracy via port 123.”

Step 4: Reflect

Write a short paragraph: How do DHCP, DNS, and NTP each keep your network running smoothly?

Hint: One gives you an address, one translates names to IPs, and one keeps your time in sync.

🔐 Premium Lab (Optional Extension): Map Services Visually with NetScanTools Basic

If you're on Windows and want to go deeper, download NetScanTools Basic (free).

  • Use its tools to look up DNS records, check NTP servers, or trace routes to services like Google or your DNS provider.
  • This helps visualize the service endpoints you explored in Command Prompt.
  • Bonus: Try a DNS brute force or TTL check to simulate advanced diagnostics.

Lab Summary: Today, you used built-in commands to inspect key network services—getting hands-on with how your computer gets an IP address, translates website names, and stays on time. These services are invisible but critical to daily internet usage.
