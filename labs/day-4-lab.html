Subnetting Basics - Mastering Class C Subnetting

Break down a /24 network into 4 subnets by hand, then verify with a tool.

Objective

Practice subnetting a Class C network by hand and verify with a tool, mastering Network+ subnetting skills.

Tools Needed

  • Windows Calculator (Programmer mode)
  • Optional: Subnet Calculator
  • Paper/pen or Notepad

Step 1: Understand Your Network

Start with Class C: 192.168.1.0/24 (255.255.255.0). Divide it into 4 subnets.

Target: IT, Sales, HR, Guest

Note: /24 = 256 addresses → 4 subnets = 64 each

Step 2: Calculate Subnets by Hand

  • Borrow 2 bits → /26 mask = 255.255.255.192
  • Addresses per subnet = 64

Ranges:

  • Subnet 1: 192.168.1.0 – .63 → Usable: .1 – .62
  • Subnet 2: 192.168.1.64 – .127 → Usable: .65 – .126
  • Subnet 3: 192.168.1.128 – .191 → Usable: .129 – .190
  • Subnet 4: 192.168.1.192 – .255 → Usable: .193 – .254

Binary mask check: 11000000 = 192

Step 3: Verify with a Tool

Visit Subnet Calculator or calculate manually.

Confirm the same four /26 ranges and subnet sizes.

  • IT: 192.168.1.0/26 → Example: Server at .10
  • Sales: 192.168.1.64/26 → Example: Laptop at .70
  • HR: 192.168.1.128/26 → Example: Printer at .130
  • Guest: 192.168.1.192/26 → Example: Guest phone at .200
