Day 4 Lab

Subnetting Basics — Mastering Class C Subnetting

Objective: Break down a /24 network into four /26 subnets by hand, then verify with a subnet calculator.

Tools Needed: Windows Calculator (Programmer mode), optional Subnet Calculator, paper/pen or Notepad.

Step 1: Understand Your Network

Start with Class C: 192.168.1.0/24 (mask 255.255.255.0). Your goal is to create 4 equal subnets (for IT, Sales, HR, Guest).

  • /24 has 256 total addresses (0–255).
  • To make 4 subnets, borrow 2 bits from the host portion → /26.
  • /26 mask = 255.255.255.192 (binary: 11000000 → 192).
  • Addresses per subnet = 64 (62 usable hosts each).

Step 2: Calculate Subnets by Hand

Increment the 4th octet by 64 to get each network ID and range:

  • Subnet 1: Network 192.168.1.0 — Broadcast 192.168.1.63 → Usable 192.168.1.1 – 192.168.1.62
  • Subnet 2: Network 192.168.1.64 — Broadcast 192.168.1.127 → Usable 192.168.1.65 – 192.168.1.126
  • Subnet 3: Network 192.168.1.128 — Broadcast 192.168.1.191 → Usable 192.168.1.129 – 192.168.1.190
  • Subnet 4: Network 192.168.1.192 — Broadcast 192.168.1.255 → Usable 192.168.1.193 – 192.168.1.254

Step 3: Verify with a Tool

Open Subnet Calculator (or any equivalent), enter 192.168.1.0/26, and confirm the ranges above. Assign example hosts:

  • IT: 192.168.1.0/26 → e.g., Server 192.168.1.10
  • Sales: 192.168.1.64/26 → e.g., Laptop 192.168.1.70
  • HR: 192.168.1.128/26 → e.g., Printer 192.168.1.130
  • Guest: 192.168.1.192/26 → e.g., Phone 192.168.1.200

Step 4: Reflect

Write a short paragraph: What does “borrowing bits” mean? How do the network and broadcast addresses define the usable range? How did the tool confirm your math?

Hint: Borrowing 2 bits from a /24 creates a /26 (4 subnets, 62 usable hosts each). The first address is the network ID; the last is the broadcast.
